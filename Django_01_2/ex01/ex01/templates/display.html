Ex01: Display process of a static page

The process of displaying a static web page in Django involves several key steps, from the user's request to the final response sent by the server. Here is a summary of the main stages:

  1. 1. The user sends a request:
    The process begins when a user enters a URL (for example, /ex01/display) in their browser and sends an HTTP request to the Django server.
  2. 2. URL routing:
    Django checks the urls.py file to find which view corresponds to the requested URL. If the URL matches, Django calls the associated view function.
  3. 3. The view processes the request:
    The view function (in views.py) receives the request and determines which template should be rendered. For a static page, the view simply calls render() with the template name (e.g., display.html).
  4. 4. Template rendering:
    Django loads the specified template and processes it. If context variables are provided, they are inserted into the template; for a static page, the template is usually rendered as-is.
  5. 5. Sending the response:
    The rendered HTML page is returned as an HTTP response to the user's browser, which then displays the static page.

This process allows Django to efficiently serve static or dynamic content using its powerful template and view system.
