Ex01: Template engine

Django's default template engine is a powerful system for generating dynamic HTML pages. It allows you to separate the presentation (HTML) from the logic (Python code), making your code cleaner and easier to maintain.

How the template engine works

When a view calls render(), Django loads the specified template, processes its tags and variables, and returns the final HTML to the browser. Templates can include logic such as blocks, loops, and conditions.

Blocks ({% block %})

Blocks are used in template inheritance. The base.html template defines blocks (e.g., {% block content %}), and child templates can override these blocks to insert their own content. 


{% block content %}
    <h1>Hello!</h1>
{% endblock %}

For loops ({% for ... in ... %})

The for tag lets you iterate over a list or queryset passed from the view. 


{% for item in items %}
    <p>{{ item }}</p>
{% endfor %}

If statements ({% if ... %})

The if tag allows you to add conditional logic in your templates. 


{% if user.is_authenticated %}
    <p>Welcome, {{ user.username }}!</p>
{% else %}
    <p>Hello, guest!</p>
{% endif %}

Displaying context variables

Variables passed from the view to the template can be displayed using double curly braces {{ variable }}. 


{{ variable }}

These features make Django templates flexible and powerful for building dynamic web pages.
